# 🚀 **Advanced React Feature Learning Path (2026 Edition)**

## **PHASE 1 — React Under the Hood (Core Internals)**

**Goal:** Understand how React _actually works_ so advanced features make sense.

### 🔥 Topics

- **Reconciliation & Fiber architecture**
    - How React schedules work
    - Concurrent rendering fundamentals
- **Rendering lifecycle**
    - Commit vs render phases
    - Why effects run twice in Strict Mode
- **React performance model**
    - Batching
    - Memoization boundaries
    - React’s event system

### 🧪 Project

Build a “Render Visualizer” that shows when components re-render and why.

### ✔️ Mastery Check

Explain why React sometimes re-renders components even when props don’t change.

---

## **PHASE 2 — Advanced Hooks & Patterns**

**Goal:** Move beyond useState/useEffect into scalable patterns.

### 🔥 Topics

- **useReducer + context for complex state**
- **Custom hooks architecture**
- **Effect orchestration**
    - Avoiding race conditions
    - AbortController patterns
- **useCallback / useMemo — when they actually help**
- **useLayoutEffect vs useEffect**
- **useSyncExternalStore** for external state libraries

### 🧪 Project

Create a reusable “Form Engine” using custom hooks (validation, async rules, debouncing).

### ✔️ Mastery Check

Refactor a component with 5+ effects into a single, predictable custom hook.

---

## **PHASE 3 — State Management at Scale**

**Goal:** Learn modern state management that works with React 18+.

### 🔥 Topics

- **Redux Toolkit (RTK)**
- **Zustand** (simple, scalable)
- **Jotai / Recoil** (atomic state)
- **Server state vs client state**
- **React Query / TanStack Query**
    - Caching
    - Background refetching
    - Optimistic updates

### 🧪 Project

Build a “Dashboard with Live Data” using React Query + Zustand.

### ✔️ Mastery Check

Explain the difference between _server state_, _client state_, and _derived state_.
